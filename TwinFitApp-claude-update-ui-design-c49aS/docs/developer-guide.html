What is TwinFit?

A paired fitness app where two people keep each other accountable. Both must check in daily to maintain their shared streak. Core loop: photo check-in → streak → quests → dumbbell economy → shop rewards.

🔥
Duo Streaks
Both partners log the same day or the streak resets. Drives the entire engagement loop. Personal + group streaks tracked separately.
📸
Photo Check-ins
Camera check-ins uploaded to Supabase Storage CDN. Public URL stored in DB so photos persist across devices and reinstalls.
🧬
AI Nutrition
Voice or text meal description → Claude Haiku returns calories, protein, carbs, fat, healthScore. Also generates full recipes with steps and macros.
🏋️
Dumbbell Economy
In-app currency earned via quests. Spend on XP boosts (2–3×), avatar borders (15 tiers from Iron to Apex Sovereign), and packs.
⚡
Evolution Tiers
Bronze(0–19) → Silver(20–49) → Gold(50–99) → Platinum(100–199) → Legendary(200+). Animated tier nodes with glow on active tier.
🔔
Push Notifications
Expo push tokens saved to backend on login. Streak reminders scheduled at 8PM daily via local notifications. Android channels configured.
Tech Stack

Every layer of the stack with version and rationale.

📱
React Native + Expo SDK 52
New Architecture (Fabric). Expo Router NOT used — uses React Navigation native stack + bottom tabs. Full TypeScript. expo-secure-store for token persistence.
🖥️
Next.js 15 App Router
Pure API backend — no pages rendered. All handlers in backend/app/api/**/route.ts. All routes dynamic (ƒ). Deployed as Vercel serverless functions.
🗄️
Prisma 7 + Accelerate
PostgreSQL via Prisma Accelerate (connection pooling + edge cache). URL format: prisma+postgres://. Lazy-proxy pattern for Vercel build compatibility.
🟩
Supabase
PostgreSQL host + Storage for check-in photos. Bucket: checkins (public, 5MB, JPEG/PNG/WEBP). RLS policies for SELECT + INSERT.
🤖
Anthropic Claude Haiku
Model: claude-haiku-4-5-20251001. Called via raw fetch from Next.js route handlers. Used for meal scan + recipe generation.
🔐
JWT Auth (7-day)
Signed with jsonwebtoken. Stored in expo-secure-store. Sent as Bearer header. Verified via getUserId(req) in every protected route.

Auth Flow

JWT-based, stateless backend. Token lives in expo-secure-store on device.

backend/lib/auth.tsTypeScript
const jwtSecret = process.env.JWT_SECRET || "dev_secret";

export function getUserId(req: NextRequest): string | null {
  const header = req.headers.get("authorization");
  if (!header) return null;
  const token = header.replace("Bearer ", "");
  try {
    const payload = jwt.verify(token, jwtSecret) as { sub: string };
    return payload.sub;
  } catch { return null; }
}
export function signToken(userId: string) {
  return jwt.sign({ sub: userId }, jwtSecret, { expiresIn: "7d" });
}
Pattern — every protected routeTypeScript
export async function GET(req: NextRequest) {
  const userId = getUserId(req);
  if (!userId) return unauthorized();  // 401
  const data = await prisma.model.findMany({ where: { userId } });
  return Response.json({ data });
}
src/store/AuthStore.tsx — applyAuth()TypeScript
const applyAuth = async (res: AuthResponse) => {
  setToken(res.token);              // in-memory for apiFetch headers
  await storeToken(res.token);       // SecureStore — persists across restarts
  await fetchAndSetUser(res.token);  // pulls full profile (emoji, bio...)
  registerPushToken().catch(() => {}); // non-blocking
};

State Management

Two React Context providers. No Redux. No Zustand. Auth = useState. Economy = useReducer.

🔐
useAuth()
Returns: user, token, loading, error, bootstrapped, login(), register(), logout(), refreshUser(). Token auto-loaded from SecureStore on boot.
🏋️
useDumbbells()
Returns: state.{balance, xp, quests, ownedBorders}, earn(), spend(), completeQuest(), purchaseBorder(), activateBoost(), refreshWallet(). Syncs from API on token change.

Critical — Lazy Prisma Init (why it exists)

Problem: Next.js 15 executes route modules at build time to collect metadata. new PrismaClient() at module-level throws PrismaClientInitializationError during build even when DATABASE_URL is set.

Fix: backend/lib/prisma.ts exports a Proxy that only creates PrismaClient on the first actual property access (at request time, not build time).
backend/lib/prisma.ts — the fixTypeScript
export const prisma: any = new Proxy({}, {
  get(_target, prop: string) {
    return getClient()[prop]; // lazy — first access creates PrismaClient
  },
});
// Also: removed 'driverAdapters' from previewFeatures in schema.prisma
// (it's now GA, having it caused constructor errors in Prisma 7)
